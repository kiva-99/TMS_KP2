# TMS_KP2

# DevOps Practical Exam (60 minutes)

## Сценарий

Компания запускает новый микросервис "quote-generator", но инфраструктура отсутствует. Вы DevOps engineer в этой компании. Вперед...

---

Цель
Поднять сервис с нуля с использованием:
контейнеризации
IaC
конфигурационного менеджмента
CI/CD
базовой отказоустойчивости

У вас есть:
- local machine
- доступ к cloud provider (AWS, GCP, Yandex cloud)

---

# 🖥️ Требования (Local Machine)

Перед началом, убедитесь что у вас установлены и работают:

- Git  
- Ansible  
- Terraform  
- Docker  

---

# Задания
1. Создание VM через Terraform (10 минут)

Создайте виртуальную машину в ОДНОМ облаке:
AWS
GCP
Yandex Cloud

Требования:
Открыть порты:
22 (SSH)
5000 (приложение)
8080 (Jenkins)

Ожидаемый результат:
Получен публичный IP-адрес

Важно: помните про безопасность вашего сервера и ограниченный доступ к нему!!!

2. Доступ к VM (обязательный шаг)
Подключение по SSH:
ssh -i key.pem <user>@<VM_IP>

Создать inventory для Ansible (inventory.ini):

Проверка подключения:
ansible -i inventory.ini servers -m ping

Ожидаемый результат:
SUCCESS => pong

3. Настройка сервера через Ansible (10 минут)
Создайте playbook.

Он должен:
- установить Docker
- добавить пользователя в группу docker
- запустить Jenkins в Docker контейнере

Требования:
Jenkins должен быть доступен на порту 8080

Ожидаемый результат:
http://<VM_IP>:8080

4. Контейнеризация приложения (10 минут)
Создайте:
Dockerfile

- соберите образ
- запустите контейнер

Требования:
базовый образ: python:3.11-slim
порт: 5000

Ожидаемый результат:
Приложение доступно локально или на VM

5. Отказоустойчивость (10 минут)
Создайте docker-compose.yml

Требования:
запустить 2 экземпляра приложения
добавить restart policy

Ожидаемый результат:
Сервис продолжает работать при остановке одного контейнера

6. CI/CD через Jenkins (15 минут)
Настройте pipeline в Jenkins.

Создайте Jenkinsfile со стадиями:
- Сборка Docker-образа
- Запуск контейнера и проверка /health
- Деплой через docker-compose

Ожидаемый результат:
Pipeline успешно выполняется
Приложение развёртывается через Jenkins

7. Финальная проверка (5 минут)
Должно работать:
http://<VM_IP>:5000
http://<VM_IP>:5000/health
Pipeline в Jenkins завершён успешно
